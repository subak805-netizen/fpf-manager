현재 vs 변경 후

결제 워크플로 변화

❌ 현재 (공장 중심)
1. 결제 관리 탭 들어감
2. 아이케이 카드 펼침 → 47장 원피스 행 체크
3. 백제사 카드 펼침 → 원피스 행 체크
4. 다산사 카드 펼침 → 원피스 행 체크
5. 블루패킹 카드 펼침 → 원피스 행 체크
→ 한 아이템 결제하는 데 카드 4번 펼침/이동
✅ 변경 후 (아이템 중심)
1. 결제 체크 대시보드 들어감
2. 원피스 행 체크박스 한 번
3. 4개 공장 청구액·명세서 모두 펼쳐짐
4. 각 공장 명세서 확인 → "5개 공장 전부 결제 체크" 한 번 클릭
→ 클릭 2번이면 끝

⚙️ 구현 시 영향

✅ 유지: 데이터 구조는 그대로. payFees·sch·it[ck]·it[fk] 모두 변경 없음.
🛠 신규: 결제 관리 탭에 "📋 아이템 중심" / "🏭 공장 중심" 토글 추가. 사용자가 선택해서 보기.
🔁 재사용: 거래명세서 생성(buildPayReceiptHTML)·결제 체크(paySt)는 기존 그대로. UI만 바뀜.
🎯 합산 로직 추가: 다중 선택 시 공장별 그룹화. 같은 공장이면 수량/금액 합산, 다른 공장이면 별도 행. 새 함수 1개 정도면 됨.

🤔 결정 사항 (피드백 부탁)

  1. 이 아이템 중심 view를 기존 결제관리 탭에 토글로 추가할까요, 아니면 완전 대체할까요?
    → 토글이 안전 (기존 사용자도 적응 시간)
  2. 합산 명세서 양식은? 한 공장에 묶은 통합 명세서 vs 아이템별 명세서 모음
    → 통합 추천 (공장 입장에서도 한 장으로 받는 게 편함)
  3. 출고일 없는 아이템도 표시? 출고 전이지만 단가는 알려져있는 경우
    → 출고일 있는 것만 표시 (결제 = 출고 후)
  4. 아이템 단위 vs 오더 단위: 같은 아이템이 5/2 47장, 5/9 30장 따로 출고됐다면 두 행으로 분리?
    → 출고일별로 행 분리 추천 (회차별 결제 처리)
