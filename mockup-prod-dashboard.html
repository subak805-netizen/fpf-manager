📐 디자인 핵심

  • 브랜드별 그룹. 헤더에 브랜드 컬러 pill + 진행 카운트 + "💬 브랜드에 일정 안내" 버튼
  • 한 행 = 한 아이템. 좌측 기본정보(오더일·디데이·품번·이름·컬러·수량) + 우측 6공정(재단·외주·봉제·완성·검품·출고) + 결제
  • 각 공정 셀: 공장 pill / 날짜 / 수량 / 완료 체크 / [확인] [알림] 버튼
  • 날짜: 직접 input 편집 (캘린더 popup)
  • 장수: 카드 클릭 → 모달 열림. 오더관리 회차 입력 UI 그대로 (날짜·컬러·사이즈·수량·메모)
  • 완료 토글: 체크박스 하나로 완료 상태 변경 (자동 저장)
  • 알림 버튼: 노란 카카오 컬러 — 각 공정마다 해당 공장에 카톡으로 안내
  • 출고 셀: 다크 강조 (제일 중요한 마일스톤)
  • 결제 셀: 진행 상태 pill + "결제 보기" → 결제관리 탭으로 점프
  • 디데이 색상: 지연(빨강) / D-3 이내(노랑) / 그 외(회색)
  • 외주·완성 없음: 해당 아이템이 그 공정 없으면 회색 "없음" 표시

🤔 흐름 비교

  • 오더 관리 탭: 한 아이템을 클릭 → 7개 서브탭(발주·거래처·입고·공장진행·결제서·메모). 깊이 작업.
  • 생산 대시보드 (이 화면): 모든 아이템 × 모든 공정을 한눈에. 날짜·수량 빠르게 변경, 알림 카톡.
  • "📅 일정 안내": 오더 탭의 일정 버튼 → 생산 대시보드 상단으로 이동. 브랜드별로 한 번에 보냄.
  • 결제 보기: 결제관리 탭으로 점프 (방금 만든 아이템 중심 view)
