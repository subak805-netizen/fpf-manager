비즈니스 규칙: 명세서 생성 단위

  • 1 명세서 = 1 공장 × 1 브랜드. 사업자등록·세금계산서가 브랜드별로 발행되므로 같은 공장이라도 브랜드 다르면 분리.
  • 같은 공장 × 같은 브랜드 × 여러 회차 출고 → 합산 (회차별 소계로 표시). 케이스 A 참고.
  • 같은 공장 × 다른 브랜드 → 별도 명세서 N장. 케이스 A·D 비교 참고.
  • 출고일 오름차순 정렬 강제.

디자인 정리

  • 테두리 전부 제거. 외곽·헤더 박스·표 셀 모두 line 없음. shadow와 여백으로만 영역 구분.
  • 회색 톤 단일화. #111827 (텍스트) → #6b7280 (보조) → #9ca3af (라벨) → #e5e7eb (구분선) → #f3f4f6, #fafbfc (배경) 5단계만.
  • 표 행 구분: 가로 얇은 선(#f3f4f6) 1줄만. 세로선 없음.
  • 합계 영역: 박스 없이 위 1px 회색 선 + 폰트 크기로 hierarchy. 최종 합계 20px / 700.
  • VAT 포함 배지: 파란색에서 차분한 회색 pill로.
  • 입금 계좌: 청록색 → 옅은 회색 #fafbfc. 시선 끌기보다 자연스러운 톤.
  • 액션 버튼: 컬러 강조 줄임. 이미지 저장은 dark color로 primary 표시.
  • 전체 폰트: 한글은 Apple SD Gothic Neo · Pretendard, antialiased. 자간 -0.3px 미세 조정.
